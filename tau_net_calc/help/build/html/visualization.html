10. Visualization of accessibility computations¶

ACCESSIBILITY plugin finishes every scenario by presenting the thematic map of the results. In case of the AREA option, the results are visualized for the first 5 origins in case of Forward or first 5 destination in case of Backward accessibility. In case of the MAP option, the results are visualized for the maximal travel time (largest bin), for the number of acccessible buildings and for the first 5 attributes seleted for aggregation.

10.1. The layers for visualization¶

The results are visualized based on:

  • The layer of buildings per se

  • The layer of the Voronoi polygons constructed based on the buildings’ foundations, supplied with the plugin.

  • The layers of hexagonal cells h3 at resolutions of h11, h9, h10, h11, supplied with the plugin.

The IDs of the visualization polygons must repeat the IDs of the buildings (some of them), and each polygon will present the results for the building with this ID.

10.2. The style of presentation¶

The default style is Graduated, with the Yellow-Green-Violet pallette that covers the entire [0, Maximal accessibility] interval.

The bins of the presentation are fixed:
- For the AREA computation - 5 minutes
- For the MAP computations - 5000 buildings when presenting the number of accessible buildings
- For the MAP computations - 1/10 of the maximal velue of the aggregated attribute

The number of grades is thus dependent on the range of the data
